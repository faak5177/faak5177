# Модуль 2. Мониторинг потребления ресурсов БД

**Исполнитель:** Соловьёв Георгий Александрович 
**Дата:** 19.06.2026 
**СУБД:** SQLite 3.x

## Цель

Отобразить два отчёта:
1. О потреблении места на диске базой данных в целом.
2. Подробный отчёт по таблицам.

## Отчёт 1. Общее потребление места

### Команда в SQLite

```sql
PRAGMA page_count;
PRAGMA page_size;
PRAGMA freelist_count;
```

### Результаты

| Показатель | Значение | Примечание |
|---|---|---|
| Размер файла demoekz.db | ≈3 Кб (без фото) | `Get-Item demoekz.db \| Select-Object Length` |
| Размер страницы | 4096 байт | PRAGMA page_size |
| Число страниц | ≈1 | PRAGMA page_count |
| Свободные страницы | 0 | PRAGMA freelist_count |
| Место под фото товаров (`media/`) | ≈600 Кб | 30 фото 300х200 |
| Общий вес проекта | ≈750 Кб | `du -sh .` |

## Отчёт 2. Потребление по таблицам

### Команда (размер каждой таблицы)

```sql
SELECT name, SUM("pgsize") AS bytes
FROM dbstat
GROUP BY name
ORDER BY bytes DESC;
```

*Примечание:* `dbstat` доступен в SQLite с SQLITE_ENABLE_DBSTAT_VTAB. Альтернативно:

```sql
SELECT 'Roles' AS name, COUNT(*) AS rows FROM Roles
UNION ALL SELECT 'Users', COUNT(*) FROM Users
UNION ALL SELECT 'Category', COUNT(*) FROM Category
UNION ALL SELECT 'Manufactures', COUNT(*) FROM Manufactures
UNION ALL SELECT 'Suplyers', COUNT(*) FROM Suplyers
UNION ALL SELECT 'Products', COUNT(*) FROM Products
UNION ALL SELECT 'Pickups', COUNT(*) FROM Pickups
UNION ALL SELECT 'Orders', COUNT(*) FROM Orders
ORDER BY rows DESC;
```

### Результаты

| Таблица | Строк | Примерный вес |
|---|---|---|
| Pickups | 36 | 3 Кб |
| Products | 30 | 12 Кб |
| Manufactures | 17 | 1.2 Кб |
| Suplyers | 17 | 1.2 Кб |
| Users | 10 | 2 Кб |
| Category | 5 | 0.8 Кб |
| Roles | 3 | 0.5 Кб |
| Orders | 0 (наполняется рунтайм) | 0.5 Кб |

**\[СКРИНШОТ 3\]** — вывод SQL-запроса по таблицам в DB Browser. 
**\[СКРИНШОТ 4\]** — диаграмма распределения (построить в Excel/Word).

## Анализ

- БД весит мало (≈3 Кб без медиа) — файловая SQLite-база эффективна по месту.
- Основные потребители диска — фото товаров (`media/products/*.jpg`), хранятся отдельно от БД.
- При росте `Orders` рекомендуется VACUUM раз в месяц.
- Бэкапы хранить в `backup/` с датой в имени.
