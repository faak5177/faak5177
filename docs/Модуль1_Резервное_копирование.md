## Цель

Выполнить процедуру резервного копирования БД в выбранной СУБД (SQLite) для минимизации риска потери данных и восстановления работоспособности системы в критические сроки.

## Шаг 1. Подготовка окружения

Исходная БД: `demoekz.db` (в корне проекта).
Схема: 8 таблиц (Roles, Users, Category, Manufactures, Suplyers, Products, Pickups, Orders).

**\[СКРИНШОТ 1\]** — окно DB Browser с открытой БД demoekz.db, видны таблицы.

## Шаг 2. Создание резервной копии

### Способ А. Через утилиту `sqlite3` (CLI)

```bash
sqlite3 demoekz.db ".dump" > backup/User01_РКБД_Соловьёв_01-bc01.sql
```

**\[СКРИНШОТ 2\]** — приглашение PowerShell/Terminal с выполненной командой.

### Способ Б. Через DB Browser for SQLite (GUI)

1. File → Export → Database to SQL file...
2. Выбрать все таблицы.
3. Отметить: Keep original schema, Multiple INSERT statements.
4. Сохранить как `User01_РКБД_Соловьёв_01-bc01.sql`.

**\[СКРИНШОТ 3\]** — окно экспорта в DB Browser. 
**\[СКРИНШОТ 4\]** — проводник с файлом бэкапа в папке backup/.

## Шаг 3. Проверка бэкапа

```bash
sqlite3 demoekz_restored.db < backup/User01_РКБД_Соловьёв_01-bc01.sql
sqlite3 demoekz_restored.db "SELECT COUNT(*) FROM Products;"
```

Ожидаемый результат: 30 (товаров).

**\[СКРИНШОТ 5\]** — вывод команды, подтверждающий восстановление.

## Инструкция по сборке DOCX

1. Открыть этот markdown-файл в Word (или скопировать текст).
2. Вставить 5 скриншотов на места, помеченные `\[СКРИНШОТ N\]`.
3. Сохранить как `Модуль1_РКБД_Соловьёв.docx`.
4. Файл → Экспорт → PDF → `Модуль1_РКБД_Соловьёв.pdf`.
5. Залить оба файла в `docs/` через GitHub → Add file → Upload files.
